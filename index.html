Problem / Why VASAE

SAE feature names are usually assigned after training.

VASAE asks whether part of feature naming can move into training without breaking reconstruction.

Problem

Post-hoc names are useful but indirect.

A standard SAE learns sparse dictionary directions first. Researchers then inspect top-activating contexts or use automated explanation tools to name those directions.

Move

Use the vocabulary as a geometric anchor set.

VASAE keeps dictionary directions learnable while adding a soft objective that pulls them toward fixed token embeddings.

Scope

The name is an anchor, not an explanation.

A token name is the nearest embedding anchor for a learned direction. It is a compact label for checking evidence, not a full feature interpretation.

Method Snapshot

Standard SAE names are post-hoc; VASAE makes nearest-token names available during training.

Claim Boundary

Read token names as anchors, not explanations.

Meaning

Nearest token embedding to a learned SAE dictionary direction.

An intrinsic token name is a vocabulary-level geometric label. It helps locate and compare feature directions, but it does not complete the interpretation.

Does not mean
  • The feature has a complete semantic explanation.
  • The model causally uses that token or concept.
  • Intervening on the feature controls the named token.
  • Each feature has a unique one-to-one token name.
